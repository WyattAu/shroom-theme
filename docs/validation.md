# Export Format Validation Report

This document records the validation status of each export format generated by tools/convert.js.

## Validation Method

Each exported file is validated both automatically (CI) and manually:
- **Automated:** CI checks for valid syntax (XML well-formedness, JSON parse, YAML structure, TOML structure, CSS structure)
- **Manual:** Visual inspection in the target application

## Automated CI Validation

The following checks run on every push via `.github/workflows/ci.yml`:

- tmTheme: contains `<?xml` header
- .icls: contains `<scheme` root element
- .vim: contains `set background` directive
- Windows Terminal JSON: valid JSON via `python3 -c "import json; json.load(open('$f'))"`
- .itermcolors: contains `<?xml` header
- .yaml: contains `background:` key
- .toml: contains `[colors.primary]` section
- .conf: contains `background ` directive (with trailing space)
- .css: contains `:root` selector
